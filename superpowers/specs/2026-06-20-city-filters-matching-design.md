# Дизайн: город, фильтры ленты и улучшение алгоритма

## Контекст

Telegram-бот знакомств. Уже реализованы: регистрация, профиль, лайки, взаимные лайки, лента анкет с базовым скорингом, настройки уведомлений.

Цель — повысить качество подбора анкет, добавив город пользователя, пользовательские фильтры и небольшой бонус совместимости за совпадение города.

## Scope

- Добавить поле `city` в анкету.
- Добавить шаг ввода города в регистрацию с мягкой валидацией.
- Добавить фильтры ленты в настройки: возрастной диапазон и «только мой город».
- Обновить алгоритм подбора: применять фильтры как жёсткие, давать небольшой бонус за совпадение города.
- Обновить отображение профиля (показывать город).
- Добавить/обновить unit-тесты.

## Out of scope

- Геолокация / расстояние между городами.
- Внешние API для валидации городов (DaData и т.п.).
- Фильтр по цели знакомства или полу.
- Автоматический подбор «похожих городов».

## Изменения в данных

### Таблица `users`

Добавить колонки:

```sql
ALTER TABLE users ADD COLUMN city TEXT;
ALTER TABLE users ADD COLUMN filter_min_age INTEGER DEFAULT 16;
ALTER TABLE users ADD COLUMN filter_max_age INTEGER DEFAULT 100;
ALTER TABLE users ADD COLUMN filter_only_my_city INTEGER DEFAULT 0;
```

- `city` — нормализованное название города (title case) или `NULL`.
- `filter_min_age` / `filter_max_age` — возрастной диапазон для ленты.
- `filter_only_my_city` — `1` = показывать только анкеты из своего города, `0` = любой город.

### Функции доступа к данным

Добавить в `database.py`:

- `update_user_city(user_id, city)`
- `update_user_filters(user_id, min_age, max_age, only_my_city)`
- `get_user_filters(user_id) -> dict`

## Регистрация

После шага с интересами добавить шаг `Registration.city`.

### Сообщение бота

```
🏙️ Введи свой город (Россия):
```

### Валидация

Принимаем ввод, если:

- длина от 3 до 50 символов;
- содержит только буквы (русские/латинские), пробелы и дефисы;
- не содержит цифр;
- не состоит из одних пробелов/дефисов;
- не попадает в базовый чёрный список спам-слов (например, «фффф», «asdf» и т.п.).

При ошибке:

```
⚠️ Название города не похоже на настоящее. Введи город ещё раз (только буквы).
```

При успехе нормализуем: убираем лишние пробелы, приводим к виду «Москва», «Санкт-Петербург».

## Настройки

В меню настроек (`handlers/settings.py`) добавить кнопку «🔍 Фильтры ленты».

### Экран фильтров

```
<b>🔍 Фильтры ленты</b>

Возраст: от 18 до 35
Город: только мой / любой
```

Клавиатура:

- «➖ Мин возраст» / «➕ Мин возраст»
- «➖ Макс возраст» / «➕ Макс возраст»
- «🏙️ Только мой город» / «🌍 Любой город» (переключение)
- «↩️ Сбросить» (min=16, max=100, only_my_city=0)
- «🔙 Назад в настройки»

### Обработка границ

- Минимальный возраст не ниже `MIN_AGE` (16).
- Максимальный возраст не выше `MAX_AGE` (100).
- Если `min > max` после изменения — автоматически выравнивать: `max = min`.

## Алгоритм подбора

### Жёсткие фильтры (`services/matching.filter_candidates`)

Текущие фильтры:

- исключить себя и уже просмотренные анкеты;
- совместимость по gender / looking_for;
- разница в возрасте ≤ 5 лет.

Добавить:

- `candidate.age` в диапазоне `[filter_min_age, filter_max_age]` пользователя.
- Убрать жёсткое ограничение «разница в возрасте ≤ 5 лет»: теперь этот диапазон контролирует пользователь через фильтры.
- Если `filter_only_my_city = 1` и у пользователя задан город — `candidate.city` должен совпадать с `user.city` (сравнение без учёта регистра). Если город пользователя не задан, фильтр «только мой город» не применяется (как будто выключен).

### Скоринг (`services/matching.score_candidate`)

Сохранить текущие веса:

- интересы — 40%;
- возраст — 30%;
- цель — 30%.

Добавить бонус:

- если `user.city` и `candidate.city` совпадают (case-insensitive), добавить **+10** к итоговому скору (0–100).
- +10 — достаточно, чтобы поднять совпадающий город в ленте, но недостаточно, чтобы перебить сильные различия по интересам/цели.

## Обработка ошибок и edge cases

1. **Старые пользователи без города**: `city = NULL`. Фильтр «только мой город» не применяется (считается, что город не задан). Город не отображается в анкете.
2. **Город задан у одного, но не у другого**: бонус не начисляется.
3. **Некорректный диапазон возраста**: автоматически выравнивать границы.
4. **Валидация города**: не гарантирует реальность названия, но отсекает явный мусор.

## Тесты

### Новые тесты

1. **Валидация города**
   - «Москва» — OK.
   - «москва» → нормализуется в «Москва».
   - «123» — отклонено.
   - «ff» — отклонено.
   - пустая строка — отклонено.
   - 51 символ — отклонено.

2. **Фильтры**
   - Кандидат младше `filter_min_age` — исключён.
   - Кандидат старше `filter_max_age` — исключён.
   - `filter_only_my_city = 1`, город совпадает — оставлен.
   - `filter_only_my_city = 1`, город не совпадает — исключён.

3. **Скоринг**
   - Два одинаковых профиля, но у одного город совпадает — у него скор выше ровно на 10.
