# Дизайн: город в ленте и модерация имён/городов

## Контекст

Telegram-бот знакомств. Уже реализованы: регистрация с вводом города, фильтры ленты, алгоритм совместимости, отображение города в личной анкете.

Цель — показывать город и в ленте анкет, а также отсеивать маты и оскорбительные слова в имени и городе при регистрации.

## Scope

- Добавить город в карточку кандидата в ленте (`format_browse_card`).
- Создать сервис модерации `services/moderation.py`.
- Проверять имя и город на маты/плохие слова по встроенному списку с учётом границ слов.
- Подключить проверку к регистрации (шаги `name` и `city`).
- Добавить unit-тесты.

## Out of scope

- Редактирование профиля/города после регистрации (если появится позже — подключим отдельно).
- Сложная антиобфускация (leet, разделённые символы, транслитерация).
- Внешние API модерации.
- Модерация увлечений, цели и других полей.

## Модерация имён и городов

### Подход

Встроенный список запрещённых слов на русском и английском без внешних зависимостей. Проверка по границам слов (`\b`), чтобы снизить ложные срабатывания на именах/городах, содержащих запрещённые подстроки.

### Функции

```python
def contains_profanity(text: str) -> bool: ...
def is_clean_name(text: str) -> bool: ...
def is_clean_city(text: str) -> bool: ...
```

`is_clean_name` и `is_clean_city` могут иметь общую логику; разделены для возможности разных списков/политик в будущем.

### Нормализация перед проверкой

- Привести к нижнему регистру.
- Схлопнуть повторяющиеся пробелы и дефисы.
- Убрать крайние пробелы/дефисы.

### Список слов

Начальный список (русский + английский): распространённые маты, оскорбления и провокационные слова. Хранится как константа в `services/moderation.py`. Можно дополнять по мере необходимости.

### Обработка нарушений

**При регистрации имени:**

```
⚠️ Имя содержит недопустимые слова. Введи другое имя.
```

**При регистрации города:**

```
⚠️ Название города содержит недопустимые слова. Введи город ещё раз.
```

## Отображение города в ленте

В `services/profile.py` функция `format_browse_card` добавляет строку:

```
📍 Город: Москва
```

Если город не задан — строка не отображается (как в `format_profile`).

## Edge cases

1. **Пользователь без города** — карточка в ленте не показывает город (поведение не ломается).
2. **Имя/город с большими буквами** — проверка case-insensitive.
3. **Слово, которое является частью нормального имени/города** — граница слова (`\b`) снижает риск false positive.
4. **Пустой ввод** — обрабатывается на уровне валидации имени/города, до модерации.

## Тесты

### Модерация (`tests/test_moderation.py`)

- Обычные имена (`Анна`, `Ivan`) — чистые.
- Обычные города (`Москва`, `London`) — чистые.
- Мат на русском — отклонён.
- Мат на английском — отклонён.
- Слово с заглавной буквы — отклонён (case-insensitive).
- Слово с лишними пробелами — отклонён (после нормализации).
- Имя/город, содержащие запрещённую подстроку, но не целое слово — допущены (например, `Dickens` не должен ловиться, если в списке `dick`).

### Город в ленте (`tests/test_profile.py`)

- `format_browse_card` включает город, если он задан.
- `format_browse_card` не включает город, если он `None`.

### Регистрация (`tests/test_registration.py`)

- Грязное имя — бот просит ввести заново.
- Грязный город — бот просит ввести заново.
