# Дизайн админ-панели бота знакомств

## Цель
Добавить в Telegram-бот команду `/admin`, доступную только владельцу (`OWNER_ID`), с полным набором инструментов: баны, модерация анкет, жалобы, рассылка, статистика, экспорт и управление интересами.

## Формат
Inline-меню aiogram 3. Никакого отдельного веб-интерфейса — всё управление происходит через кнопки и диалоги внутри Telegram.

## Авторизация
- `config.OWNER_ID = 8241460494`.
- Команда `/admin` проверяет `message.from_user.id == OWNER_ID`.
- Любой другой пользователь получает «Нет доступа».

## Разделы меню
1. **📊 Статистика** — цифры и экспорт.
2. **👤 Пользователи** — поиск по `user_id` или `@username`, просмотр анкеты, бан, удаление.
3. **🚨 Жалобы** — список жалоб от пользователей, кнопки «Забанить / Отклонить / Посмотреть анкету».
4. **📢 Рассылка** — массовая отправка сообщения всем не-забаненным пользователям.
5. **🏷 Интересы** — добавление/удаление категорий и интересов для Mini App.
6. **🚫 Баны** — быстрый ввод ID для бана/разбана.

## Баны
- Колонка `users.is_banned INTEGER DEFAULT 0`.
- `BanMiddleware` блокирует сообщения и callback-и забаненных пользователей (кроме владельца).
- Web API (`/api/me`, `/api/feed`, `/api/likes`, `/api/settings`, `/api/upload-photo`, `/api/report`) возвращает `403` для забаненных.

## Жалобы
- Таблица `reports`.
- В Mini App на карточке профиля кнопка «Пожаловаться» → `POST /api/report`.
- Админ видит открытые жалобы и решает их.

## Статистика и экспорт
- Всего пользователей, новые за сутки/неделю/месяц, лайки, просмотры, активные пользователи.
- Кнопка «Экспорт JSON» отправляет файл с `users`, `stats`, `reports`.

## Рассылка
- FSM `AdminBroadcast`: ввод текста → предпросмотр → подтверждение.
- Отправка всем не-забаненным пользователям.
- Сохраняется количество доставленных сообщений.

## Управление интересами
- Таблица `interests`.
- При старте, если таблица пуста, заполняется из `config.INTEREST_CATEGORIES`.
- `GET /api/interests` отдаёт данные из таблицы.
- Админ добавляет/удаляет категории и интересы через inline-меню.

## Логи админа
- Таблица `admin_logs`.
- Каждое действие админа (бан, удаление, рассылка, решение жалобы) записывается в лог.

## Архитектура
- `handlers/admin.py` — все handler-ы админки.
- `services/admin.py` — бизнес-логика, статистика, вспомогательные функции.
- `middlewares/ban.py` — `BanMiddleware`.
- `keyboards.py` — клавиатуры админки.
- `database.py` — новые таблицы и DAO-функции.
- `web_routes.py` — `POST /api/report` и проверка бана на защищённых endpoint-ах.
- `tests/test_admin.py` — юнит- и интеграционные тесты.

## Тестирование
- pytest.
- Моки для `Bot` и `Message`/`CallbackQuery`.
- aiohttp test client для web endpoint-ов.
- Проверка: доступ только для владельца, бан middleware, статистика, жалобы, рассылка, CRUD интересов.
