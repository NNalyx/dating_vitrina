# Дизайн: редизайн мини-приложения + загрузка фото

## Контекст

Мини-приложение Telegram (`docs/`) развёрнуто на GitHub Pages. Пользователь хочет:
1. Убрать красный цвет из интерфейса.
2. Использовать палитру тёмно-серый / чёрный / белый.
3. Добавить анимации (переходы экранов, нажатия кнопок, появление элементов).
4. Добавить возможность загрузить фото профиля в мини-приложении.
5. Сделать интерфейс более «красивым» и отполированным.

## Цель

Обновить визуальный стиль и UX регистрации, добавить загрузку фото через бэкенд, сохранив существующую логику.

## Решение

### Цветовая схема

- Фон: `#000000`
- Поверхности (кнопки-опции, поля ввода, карточки): `#1c1c1e`
- Текст на тёмном фоне: `#ffffff`
- Текст на светлых поверхностях: `#000000`
- Акцент/selected: `#ffffff`
- Вторичный текст/плейсхолдеры: `rgba(255,255,255,0.6)`
- Ошибки: `#ffffff` или светло-серый
- Красный убирается полностью.

### Анимации

- **Переход между шагами регистрации**: новый экран плавно появляется (`opacity` 0→1 + `transform: translateY(10px)` → `translateY(0)`), длительность 250–300 мс, ease-out.
- **Нажатие кнопок**: `transform: scale(0.97)` + плавная смена фона/цвета 200 мс.
- **Появление списков**: stagger-анимация для чипсов интересов и кнопок-опций с задержкой ~50 мс между элементами.
- **Фокус полей**: плавное появление белой рамки (`box-shadow` или `border-color`) 200 мс.

### Загрузка фото

- На шаге «Фото профиля» добавляется:
  - Превью выбранного изображения.
  - Кнопка «Выбрать фото» (скрытый `<input type="file" accept="image/*">`).
  - Кнопка «Пропустить».
- После выбора файл отправляется на новый эндпоинт `POST /api/upload-photo` (multipart/form-data).
- Бэкенд:
  - Валидирует `initData` из заголовка `X-Init-Data`.
  - Проверяет MIME-тип (только `image/*`) и размер файла (до 5 МБ).
  - Отправляет фото ботом пользователю (`bot.send_photo`).
  - Удаляет отправленное сообщение (`bot.delete_message`).
  - Возвращает `file_id` из ответа Telegram.
- Frontend сохраняет `photo_file_id` и передаёт его при `POST /api/register`.
- Загрузка фото остаётся опциональной.

### Бэкенд

- Передать инстанс `Bot` в `web_app.create_app(bot)` и сохранить в `app['bot']`.
- Добавить POST `/api/upload-photo` в `web_routes.py`.
- Обновить CORS-заголовки: разрешить `Content-Type` (уже есть) и multipart запросы.

### Полировка

- Увеличить скругления до `20px`.
- Единые отступы `16px`.
- Все заголовки, поля, кнопки и ошибки центрированы.
- Плейсхолдеры светло-серые, по центру.

## Файлы

- `docs/css/style.css` — цвета, анимации, полировка.
- `docs/js/screens/registration.js` — анимации переходов, загрузка фото, selected-состояние.
- `docs/js/api.js` — метод `uploadPhoto(file)`.
- `web_app.py` — передача бота в приложение.
- `web_routes.py` — эндпоинт `/api/upload-photo`.
- `znakomstvabot.py` — передача бота в `create_app`.

## Критерии приёмки

- Интерфейс использует только чёрный, тёмно-серый и белый цвета; красного нет.
- Переходы между шагами регистрации анимированы.
- Нажатия кнопок имеют визуальный отклик.
- Чипсы интересов и кнопки-опции появляются с плавной stagger-анимацией.
- Можно выбрать фото на шаге «Фото профиля», увидеть превью и успешно завершить регистрацию.
- Кнопка «Пропустить» продолжает работать.
- Бэкенд-тесты проходят.
