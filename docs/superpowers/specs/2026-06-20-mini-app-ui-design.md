# Дизайн: улучшение UI мини-приложения

## Контекст

Мини-приложение Telegram (`docs/`) уже развёрнуто на GitHub Pages. Пользователь отметил три проблемы:
1. Поля и кнопки выглядят смещёнными влево, а не по центру.
2. Текущая цветовая гамма (тёмно-серый + фиолетово-красный градиент) не нравится.
3. Кнопки выбора (пол, кого ищешь, цель) не подсвечиваются при выборе.

## Цель

Исправить вёрстку и стили минимальными изменениями, сохранив существующую логику регистрации.

## Решение

### Подход

Быстрая правка CSS + небольшой патч JS.

### Файлы

- `docs/css/style.css`
- `docs/js/screens/registration.js`

### Цветовая схема

- Фон приложения: `#000000` (чёрный)
- Поверхности (кнопки, поля ввода, карточки): `#ffffff` (белый)
- Текст на чёрном фоне и на акцентных элементах: `#ffffff`
- Текст на белых поверхностях: `#000000`
- Акцент/выделение: `#ff0000` (красный)
- Фиолетовый градиент убирается.

### Выравнивание

- Контейнер `#app` остаётся с `max-width: 480px` и центрируется через `margin: 0 auto`.
- Кнопки-опции `.option` растягиваются на всю ширину (`width: 100%`) и выравнивают текст по центру.
- Поля ввода `input`/`select` центрируют текст и placeholder.

### Selected-состояние кнопок

- В `registration.js` переключение выбора реализуется через класс `.selected` вместо инлайнового `borderColor`.
- `.option.selected` получает красный фон и белый текст.
- При выборе новой опции класс снимается со всех соседних кнопок.

## Критерии приёмки

- Все поля и кнопки выровнены по центру экрана.
- Интерфейс использует чёрно-белую гамму с красным акцентом.
- При нажатии на кнопки «Парень», «Девушка», «Парней», «Девушек», «Отношения», «Дружба», «Флирт» выбранный вариант визуально подсвечивается красным.
- Логика регистрации и валидация не меняются.
