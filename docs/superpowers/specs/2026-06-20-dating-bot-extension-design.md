# Расширение Telegram-бота знакомств — Design Spec

## Цель

Добавить в существующего aiogram-бота знакомств полноценный пост-регистрационный функционал: главное меню со статистикой, просмотр и редактирование своей анкеты, ленту анкет с алгоритмом совместимости в процентах, систему лайков и взаимных лайков с переходом в личные сообщения.

## Текущее состояние

- Бот на `aiogram` 3.x + `aiosqlite`.
- Есть регистрация: политика → возраст → имя → интересы → фото.
- База данных `dating_bot.db` будет удалена и пересоздана под новую схему.

## Архитектура

Модульный подход:

- `database.py` — чистые CRUD-функции (users, likes, views).
- `services/matching.py` — фильтрация кандидатов и расчёт совместимости.
- `services/profile.py` — форматирование карточки анкеты.
- `handlers/registration.py` — обновлённая регистрация с новыми полями.
- `handlers/menu.py` — главное меню.
- `handlers/profile.py` — просмотр и редактирование анкеты.
- `handlers/browse.py` — лента анкет с лайками/пролистыванием.
- `handlers/likes.py` — обработка входящих лайков и взаимных лайков.
- `keyboards.py` — все inline/reply клавиатуры.
- `states.py` — FSM-состояния для регистрации и редактирования.

## Регистрация (обновлённая)

Порядок шагов после `/start`:

1. Политика конфиденциальности в `<blockquote expandable>` + кнопка «Согласен».
2. Возраст (число, ≥ 16).
3. Имя/ник (минимум 2 символа).
4. Пол (кнопки: Парень / Девушка / Другое).
5. Кого ищешь (кнопки: Парней / Девушек / Всех).
6. Цель (кнопки: Отношения / Дружба / Флирт).
7. Интересы (минимум 3, inline-галочки).
8. Фото (опционально, можно пропустить).
9. Сохранение профиля + приветственное сообщение + главное меню.

## Политика конфиденциальности

Минимальная суть в разворачиваемом блоке:

```html
<blockquote expandable>
1. Мы храним: возраст, имя, пол, цель знакомства, увлечения и фото.
2. Данные используются только для подбора анкет внутри бота.
3. Бот предназначен для пользователей 16+.
4. Мы не передаём данные третьим лицам.
5. Администрация не отвечает за поведение других пользователей.
</blockquote>
```

Карточка с фото (если есть) и текстом:

```
📋 Твоя анкета

Имя: {name}
Возраст: {age}
Пол: {gender}
Ищу: {looking_for}
Цель: {goal}
Увлечения: {interests}
```

Inline-кнопки:

- ✏️ Имя
- ✏️ Возраст
- ✏️ Цель
- ✏️ Кого ищу
- ✏️ Интересы
- ✏️ Фото
- 🔙 В меню

**Ограничение:** пол редактировать нельзя.

## Лента анкет

### Фильтры

Кандидат попадает в ленту, если:

1. Это не текущий пользователь.
2. Текущий пользователь ещё не смотрел эту анкету (`views`).
3. Пол кандидата входит в `looking_for` текущего.
4. `looking_for` кандидата включает пол текущего (взаимная совместимость).
5. Цели совпадают (`goal`).
6. Возраст кандидата находится в диапазоне `[current.age - 5, current.age + 5]`.

Сортировка: по убыванию совместимости.

### Алгоритм совместимости

Общий процент складывается из трёх компонентов:

- **Общие интересы — 40%**
  - `intersection / union * 40`
- **Возрастная близость — 30%**
  - `max(0, 1 - |age1 - age2| / 10) * 30`
- **Совпадение цели — 30%**
  - Если цели совпадают: `30`, иначе `0`

Итог: сумма трёх компонентов, округляется до целого.

Пример: 5 общих интересов из 10 возможных → 20%; возраст одинаковый → 30%; цель совпадает → 30%. Итого: 80%.

### UI ленты

Карточка кандидата:

```
{name}, {age}
Совместимость: 78% ❤️

🎯 Цель: {goal}
⭐ Увлечения: {interests}
```

Inline-кнопки:

- ❤️ Лайк
- 👎 Пропустить
- 🔙 В меню

При нажатии «Пропустить» анкета записывается в `views`, чтобы не показываться повторно.

## Лайки и взаимные лайки

### Отправка лайка

При нажатии «Лайк»:

1. Записываем `likes(from_user_id=me, to_user_id=candidate)`.
2. Записываем `views(viewer_id=me, viewed_id=candidate)`.
3. Проверяем обратный лайк.
   - Если есть → это взаимный лайк:
     - Отправляем моё профиль кандидату с кнопкой «Написать» (`t.me/{username}` или user_id-ссылка).
     - Показываем мне анкету кандидата с кнопкой «Написать».
   - Если нет → просто показываем следующую анкету.

### Получение лайка

Когда пользователя лайкают, ему приходит сообщение:

```
💌 Тебя лайкнули!

{анкета лайкнувшего}
```

Inline-кнопки:

- ❤️ Лайк в ответ
- 👎 Пропустить

При «Лайк в ответ» — логика взаимного лайка: оба получают анкеты друг друга с кнопками «Написать».

### Статистика

- Отправлено лайков: `COUNT(*) FROM likes WHERE from_user_id = ?`
- Получено лайков: `COUNT(*) FROM likes WHERE to_user_id = ?`

## Оформление

- Все сообщения используют HTML-разметку.
- Эмодзи для разделов.
- Карточки анкет структурированы одинаково.
- Кнопки «В меню» доступны на ключевых экранах.

## Тестирование

- Проверка регистрации с новыми полями.
- Проверка фильтрации ленты (гендерная совместимость, возраст, цель).
- Проверка расчёта совместимости.
- Проверка взаимного лайка и отправки контактов.
- End-to-end в Telegram.

## Out of scope

- Чаты внутри бота.
- Модерация анкет.
- Геолокация.
- Telegram Mini App (будет позже).
- Premium-функции.
- Удаление анкеты.
