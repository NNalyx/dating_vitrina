# Бесконечная лента: фейковые анкеты

## Цель
Когда у пользователя заканчиваются реальные кандидаты в ленте, приложение показывает фейковые анкеты, чтобы лента не заканчивалась и пользователь продолжал взаимодействовать.

## Требования
- Фейковые анкеты визуально неотличимы от реальных: фото, имя, возраст, город, интересы, цель, описание.
- Лайк/дизлайк работает как обычно, но не создаёт мэтчей, не отправляет уведомлений и не влияет на статистику.
- После просмотра фейковая анкета больше не показывается текущему пользователю.
- Если фотографии в пуле закончились, фейки создаются без фото.
- Описания не должны часто повторяться: используется большой набор шаблонов и случайное комбинирование.

## Что не входит
- Фейки не отображаются во вкладках «Лайки» и «Мэтчи».
- Фейки не могут быть отправителями/получателями реальных жалоб.
- В первой версии не делается фоновая чистка старых фейков.

## Изменения в БД
- Добавить колонку `bio TEXT` в таблицу `users`. Пока используется только для фейков, но заложена как общее поле.
- Колонки `is_fake` и отрицательные `user_id` уже поддерживаются.

## Пул аватарок
- Папка `data/fake_avatars/` с подпапками `male/`, `female/` и `neutral/`.
- Каждый файл — заранее подготовленное реалистичное AI-лицо.
- При создании фейка:
  1. Выбирается файл, соответствующий полу анкеты.
  2. Бот загружает файл в Telegram (как при обычной загрузке фото) и получает `file_id`.
  3. `file_id` сохраняется в БД в поле `photo_file_id`.
- Если подходящая папка пуста, фейк создаётся без фото (`photo_file_id = NULL`).

## Генерация фейковой анкеты
Генератор принимает профиль текущего пользователя и создаёт кандидата, который пройдёт через `filter_candidates`:
- **Пол**: под предпочтения `looking_for` текущего пользователя.
- **Кого ищет**: так, чтобы `gender_match` проходило (обычно совпадает с полом пользователя или `all`).
- **Возраст**: в диапазоне фильтров пользователя.
- **Город**: город пользователя, если включён фильтр «только мой город»; иначе случайный город из заранее заданного списка крупных городов.
- **Цель**: с высокой вероятностью совпадает с целью пользователя.
- **Интересы**: если включён фильтр по интересам, обязательно есть пересечение с интересами пользователя; иначе случайный набор.
- **Имя**: случайное имя из списка имён по полу.
- **Описание (`bio`)**: формируется из большого набора шаблонов с подстановкой:
  - 1–2 интересов,
  - цели,
  - города,
  - случайных вводных фраз.

Примеры шаблонов:
- «Живу в {city}. Увлекаюсь {interest1} и {interest2}. Ищу {goal}».
- «Люблю {interest1}, не представляю жизни без {interest2}. Хочу {goal}».
- «{city}, {age}. Главные увлечения: {interest1}, {interest2}. Цель — {goal}».

## Интеграция с лентой
- `/api/feed` после фильтрации реальных кандидатов проверяет, остались ли подходящие фейки, не просмотренные текущим пользователем.
- Если есть — возвращает лучшего по совместимости фейка с полем `compatibility`.
- Если нет — генерирует новую партию фейков (например, 10 штук) и возвращает первого.
- Фейки попадают в тот же ответ, что и реальные кандидаты; фронт не знает разницы.

## Лайк/дизлайк фейков
- `POST /api/feed/{id}/skip` — добавляет запись в `views`, фейк не показывается снова.
- `POST /api/feed/{id}/like` — добавляет запись в `views`, **не добавляет** запись в `likes`, не проверяет взаимность, не отправляет уведомлений. Возвращает `{"status": "ok", "mutual": false}`.

## Бесконечность
- Просмотренные фейки не повторяются для одного пользователя.
- При исчерпании пула генерируется новая партия.
- Если заканчиваются фото, фейки продолжают создаваться без аватарок.

## Тестирование
- Проверить, что при отсутствии реальных кандидатов возвращается фейк.
- Проверить, что фильтры возраста/города/интересов соблюдаются.
- Проверить, что лайк фейка не создаёт мэтч и не отправляет уведомление.
- Проверить, что фейк не появляется повторно после skip/like.
- Проверить генерацию без фото, если пул пуст.

## Будущее улучшение
- Периодическая чистка фейков, не просматривавшихся более N дней, чтобы база не росла бесконечно.
- Возможность админу загружать новые аватарки.
