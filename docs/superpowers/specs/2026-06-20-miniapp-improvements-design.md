# Дизайн-документ: улучшения Mini App dating-бота

Дата: 2026-06-20

## Цель
Довести UX Telegram Mini App dating-бота до состояния, в котором старое бот-меню больше не используется, взаимные лайки дают реальный способ связаться, интересы при регистрации расширены и настройки позволяют красиво редактировать профиль.

## 1. Удаление старого бот-меню
- При `/start` для уже зарегистрированного пользователя отправлять сообщение:
  **«Регистрация успешно пройдена! Заходи в приложение»** + inline-кнопка «🚀 Открыть приложение» (WebApp).
- После завершения регистрации через Mini App (`POST /api/register`) сервер отправляет то же сообщение с кнопкой WebApp.
- Обработчики старого меню (`handlers/menu.py`, `main_menu_keyboard` в `keyboards.py`) удаляются/отключаются, чтобы меню нельзя было вызвать.

## 2. Взаимный лайк и контакт
- При взаимном лайке (бот + Mini App) формировать контакт по приоритету:
  1. `https://t.me/{username}` — если username не пустой.
  2. `tg://user?id={user_id}` — deep link по ID.
  3. Если ни один вариант не сработал — отправить анкету и текст:
     «💞 Взаимный лайк! К сожалению, не удалось получить контакт пользователя. Попробуй найти его по username в Telegram вручную.»
- Кнопка «💬 Написать» добавляется только при успешной ссылке.

## 3. Интересы
- Mini App и бот используют единый список категорий из `config.py` (`INTEREST_CATEGORIES`).
- Добавить игровые и околоигровые интересы: `Valorant`, `CS2`, `Dota 2`, `Genshin Impact`, `Fortnite`, `Minecraft`, `League of Legends`, `Apex Legends`, `Overwatch`, `osu!`, `GTA`, `Steam/PC Gaming`, `Консоли`, `Настольные игры`.
- В Mini App при регистрации и редактировании профиля интересы выбираются чипами по категориям, минимум 3.

## 4. Настройки и редактирование профиля
- Экран настроек разбит на две карточки:
  - **Моя анкета** — список полей профиля с текущими значениями. Тап открывает редактор.
  - **Фильтры и уведомления** — stepper для возраста, toggle «Только мой город», toggle «Новые лайки».
- Редакторы полей:
  - Фото — загрузка/удаление.
  - Имя, возраст, город — input с валидацией.
  - Пол, кого ищу, цель — карточки-выбор (стиль регистрации).
  - Интересы — категории чипов.
- Сохранение по кнопке «Сохранить», возврат к настройкам.

## 5. Архитектура и файлы
- `config.py` — расширить `INTEREST_CATEGORIES`.
- `handlers/common.py` — `/start` для существующих пользователей отправляет WebApp-кнопку.
- `handlers/menu.py`, `keyboards.py::main_menu_keyboard` — удалить/отключить.
- `handlers/browse.py`, `handlers/likes.py`, `web_routes.py` — улучшить логику отправки контакта при mutual like.
- `web_routes.py::register` — отправлять приветственное сообщение с WebApp-кнопкой.
- `docs/js/screens/registration.js` — чипы интересов по категориям из `config.py` (через `/api/interests` или встроенный список).
- `docs/js/screens/settings.js` — новый layout с разделами профиля и фильтров.
- `docs/js/screens/edit_*.js` — редакторы отдельных полей профиля.
- `docs/js/api.js` — добавить методы обновления профиля (PUT /api/me уже есть).

## 6. Тестирование
- Обновить/дополнить `tests/test_miniapp_features.py`:
  - проверка сообщения после регистрации;
  - mutual like с username, без username;
  - обновление профиля через API.
- Все тесты должны проходить (`pytest tests/ -v`).
